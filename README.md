# **ЗАДАЧА 2**

## Выбор платформы в зависимости от задачи

### **1. Высоконагруженная база данных MySQL, критичная к отказу**

**Рекомендуемая платформа:** Физические серверы или паравиртуализация.

**Критерии выбора:**

- **Производительность и отказоустойчивость:** Высоконагруженные базы данных требуют минимальной задержки и высокой надежности. Физические серверы обеспечивают максимальную производительность и стабильность, так как исключают накладные расходы виртуализации. Паравиртуализация может быть использована для гибкости, если требуется масштабируемость.
- **Масштабируемость:** MySQL хорошо масштабируется при правильной настройке, особенно на физических серверах с достаточным объемом оперативной памяти и быстрыми дисковыми подсистемами (например, SSD или NVMe).
- **Безопасность:** Физические серверы обеспечивают более высокий уровень контроля над данными, что важно для критичных к отказу систем.

### **2. Различные веб-приложения**

**Рекомендуемая платформа:** Виртуализация уровня ОС.

**Критерии выбора:**

- **Гибкость:** Виртуализация уровня ОС (например, контейнеры Docker) позволяет изолировать приложения, что упрощает их развертывание и управление. Это особенно полезно для веб-приложений с разными требованиями к среде выполнения.
- **Масштабируемость:** Контейнеры легко масштабируются, что важно для веб-приложений с переменной нагрузкой.
- **Удобство использования:** Современные инструменты оркестрации (например, Kubernetes) упрощают управление контейнерами и обеспечивают высокую доступность.

### **3. Windows-системы для использования бухгалтерским отделом**

**Рекомендуемая платформа:** Паравиртуализация.

**Критерии выбора:**

- **Совместимость:** Windows-системы требуют платформы, поддерживающей Windows-гостевые ОС. Паравиртуализация (например, VMware или Hyper-V) обеспечивает высокую производительность и совместимость с Windows.
- **Удобство управления:** Паравиртуализация позволяет централизованно управлять виртуальными машинами, что упрощает обслуживание и обновление систем.
- **Безопасность:** Виртуализация позволяет изолировать бухгалтерские системы от других приложений, что повышает уровень безопасности данных.

### **4. Системы, выполняющие высокопроизводительные расчёты на GPU**

**Рекомендуемая платформа:** Физические серверы.

**Критерии выбора:**

- **Производительность:** Для задач, связанных с интенсивными вычислениями на GPU (например, машинное обучение или рендеринг), физические серверы с мощными графическими ускорителями обеспечивают максимальную производительность.
- **Оптимизация под GPU:** Физические серверы позволяют эффективно использовать ресурсы GPU, минимизируя задержки передачи данных между CPU и GPU.
- **Масштабируемость:** При необходимости можно использовать кластеры серверов с GPU для распределенных вычислений.

### **Заключение**

Выбор платформы зависит от специфики задачи, доступного бюджета, требований к производительности и надежности. Для критичных систем предпочтительны физические серверы, тогда как для гибкости и удобства управления подойдут виртуализация и контейнеризация.

# **ЗАДАЧА 3**

## Выбор системы управления виртуализацией для предложенных сценариев

### Сценарий 1:

**100 виртуальных машин на базе Linux и Windows, преимущественно Windows-based инфраструктура, требуется программная балансировка нагрузки, репликация данных и автоматизированное резервное копирование.**

**Рекомендация:** **Microsoft Hyper-V**

- **Причины выбора:**
  - Hyper-V идеально подходит для Windows-based инфраструктуры, так как является нативным решением Microsoft и интегрируется с Windows Server.
  - Поддерживает программные балансировщики нагрузки и репликацию данных через встроенные инструменты, такие как Hyper-V Replica.
  - Обеспечивает автоматизированное резервное копирование через интеграцию с System Center и Windows Admin Center.
  - Простота управления и масштабируемость делают его оптимальным выбором для крупных инфраструктур.

### Сценарий 2:

**Наиболее производительное бесплатное open source-решение для виртуализации небольшой инфраструктуры (20-30 серверов) на базе Linux и Windows.**

**Рекомендация:** **Proxmox Virtual Environment**

- **Причины выбора:**
  - Proxmox VE — бесплатное open source-решение, которое поддерживает виртуализацию как на основе KVM, так и контейнеров LXC.
  - Простота установки и управления через веб-интерфейс.
  - Поддерживает Windows и Linux виртуальные машины, а также функции резервного копирования и репликации.
  - Высокая производительность и гибкость делают его идеальным для небольших инфраструктур.

### Сценарий 3:

**Бесплатное, максимально совместимое и производительное решение для виртуализации Windows-инфраструктуры.**

**Рекомендация:** **Microsoft Hyper-V Server**

- **Причины выбора:**
  - Hyper-V Server — бесплатная версия Hyper-V, специально разработанная для Windows-инфраструктуры.
  - Максимальная совместимость с Windows Server и клиентскими ОС Windows.
  - Высокая производительность и поддержка ключевых функций, таких как виртуальные сети, репликация и резервное копирование.
  - Простота интеграции с другими продуктами Microsoft, такими как System Center и Azure.

### Сценарий 4:

**Рабочее окружение для тестирования программного продукта на нескольких дистрибутивах Linux.**

**Рекомендация:** **Oracle VirtualBox**

- **Причины выбора:**
  - VirtualBox — бесплатный и кроссплатформенный гипервизор, который поддерживает широкий спектр Linux-дистрибутивов.
  - Простота настройки и возможность создания изолированных тестовых сред.
  - Поддержка снимков (snapshots), что удобно для тестирования и отката изменений.
  - Легкость в использовании и отсутствие сложных требований к аппаратному обеспечению делают его идеальным для тестовых окружений.

### Итог

Каждое из предложенных решений соответствует специфическим требованиям сценариев. Hyper-V подходит для крупных Windows-инфраструктур, Proxmox VE — для небольших смешанных сред, а VirtualBox — для тестирования.
